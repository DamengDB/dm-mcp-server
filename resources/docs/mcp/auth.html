MCP 服务配置认证与数据源访问

认证与数据源访问

本文档说明 AI 客户端连接达梦 MCP 服务时的认证方式，以及令牌与数据源之间的权限关系。

令牌与数据源的关系

达梦 MCP 的访问令牌与数据源之间是多对多的关系：

  • 一个令牌可绑定多个数据源 — 在 令牌管理 页面创建令牌时，通过”允许访问的数据源”字段选择多个数据源，令牌有权访问其列表中的每一个。
  • 一个数据源可被多个令牌引用 — 在 数据源管理 页面配置的数据源，可以被不同的令牌各自绑定，实现不同客户端或不同场景的独立权限控制。

这种设计的目的是支持最小权限原则。例如，你可以为开发环境创建一个仅绑定测试数据源的令牌，为生产运维创建另一个仅绑定生产数据源的令牌，两者互不干扰。

数据源选择优先级

当 AI 客户端发起请求时，系统按以下顺序确定使用哪个数据源：

  1. 请求头部指定 — 如果请求携带 X-DMMCP-DataSource 头部，使用其指定的数据源名称
  2. 令牌默认数据源 — 如果未指定头部，使用令牌创建时设置的默认数据源。在 令牌管理 页面创建令牌时，“默认数据源”字段必须从”允许访问的数据源”列表中选择一项
  3. 权限校验 — 最终确定的数据源必须在令牌的允许列表中，否则返回 403 错误

注意：创建令牌时必须指定至少一个数据源，否则令牌无法通过认证。默认数据源必须是允许列表中的一项。

HTTP 认证头部

Authorization（必需）

MCP 路由的 API 请求必须使用 Token 认证。访问令牌在 令牌管理 页面创建，创建后系统会自动将其复制到剪贴板。在 HTTP 请求头中携带 Authorization 字段：

Authorization: Bearer sk-dmmcp-AbCdEfGhIjKlMnOp...

X-DMMCP-DataSource（可选）

在 HTTP 请求中通过此头部动态指定要使用的数据源。数据源名称在 数据源管理 页面配置，创建时填写的”名称”字段即为此处使用的值：

X-DMMCP-DataSource: my_production_db
  • 头部值为数据源配置中的名称字段，大小写敏感
  • 指定的数据源必须在令牌的允许列表中
  • 未指定时，自动回退到令牌的默认数据源

配置示例

Claude Code

在 Claude Code 的 MCP 配置中添加达梦 MCP 服务：

{
  "mcpServers": {
    "dm-mcp": {
      "url": "http://your-server/dm-mcp/mcp",
      "headers": {
        "Authorization": "Bearer sk-dmmcp-xxxxxxxx"
      }
    }
  }
}

切换数据源

若令牌绑定了多个数据源，可以通过额外指定 X-DMMCP-DataSource 头部来切换：

{
  "mcpServers": {
    "dm-mcp-prod": {
      "url": "http://your-server/dm-mcp/mcp",
      "headers": {
        "Authorization": "Bearer sk-dmmcp-xxxxxxxx",
        "X-DMMCP-DataSource": "production_db"
      }
    },
    "dm-mcp-test": {
      "url": "http://your-server/dm-mcp/mcp",
      "headers": {
        "Authorization": "Bearer sk-dmmcp-xxxxxxxx",
        "X-DMMCP-DataSource": "test_db"
      }
    }
  }
}

上例中，同一个令牌通过不同的 MCP 服务配置分别指向生产库和测试库。

curl 测试

curl -X POST http://your-server/dm-mcp/mcp \
  -H "Authorization: Bearer sk-dmmcp-your-token" \
  -H "X-DMMCP-DataSource: my_datasource" \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","method":"tools/call","params":{"name":"get_db_schemas_list","arguments":{}},"id":1}'

IP 访问控制

在 令牌管理 页面创建或编辑令牌时，可以通过”IP 名单”开关选择白名单或黑名单模式，限制访问来源：

  • 白名单 — 仅允许列表中的 IP 地址或 CIDR 网段使用该令牌
  • 黑名单 — 禁止列表中的 IP 地址或 CIDR 网段使用该令牌

CLI 工具使用提示

DM Agent CLI 不会发送 X-DMMCP-DataSource 头部，因此每次请求都会使用令牌的默认数据源。

如果你的令牌绑定了多个数据源，建议采取以下策略之一：

  1. 在 令牌管理 页面设置合适的默认数据源 — 将 CLI 最常用的数据源设为默认。
  2. 为 CLI 工具单独创建令牌 — 可以单独创建一个仅绑定 CLI 所需数据源的令牌。
