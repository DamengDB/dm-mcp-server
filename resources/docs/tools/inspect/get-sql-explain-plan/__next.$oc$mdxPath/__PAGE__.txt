5:T19a0,# get_sql_explain_plan <ToolBadge />

对指定 SELECT 语句进行真实执行，返回执行计划及运行时统计。

返回执行计划树文本、语句级 I/O 指标、算子级耗时与资源占用（含哈希统计），以及与 Sort、Hash Join、Hash 聚合相关的会话 INI 参数。适用于慢 SQL 诊断、执行计划分析、算子级性能瓶颈定位；宜在已通过会话或 SQL 热点工具锁定目标语句后调用。

## 参数

| 名称 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| sql | str | - | 待分析的执行计划 SQL。仅支持 SELECT（含 WITH）；允许 Hint 与注释；禁止分号；长度上限 10000 字符。 |
| schema | str | None | 可选 schema；未指定时使用数据源默认值。 |

## SQL: Check Schema Exists

指定 schema 时校验其是否存在。

```sql
SELECT 1 AS X
FROM SYS.SYSOBJECTS SCH_OBJ
WHERE SCH_OBJ.TYPE$ = 'SCH' AND SCH_OBJ.NAME = ?
FETCH FIRST 1 ROWS ONLY
```

## SQL: Enable Explain Monitor

同连接上开启 SQL 执行监控（一次执行）。

```sql
BEGIN
    SF_SET_SESSION_PARA_VALUE('MONITOR_SQL_EXEC', 1);
    SF_SET_SESSION_PARA_VALUE('ENABLE_MONITOR_DMSQL', 1);
END;
```

## SQL: Get Runtime Stat By Exec Id

按 `exec_id` 合并查询语句级运行时统计（`V$SQL_STAT` / `V$SQL_STAT_HISTORY` / `V$SQL_HISTORY`，字段取非空优先）。

```sql
SELECT
    COALESCE(s.EXEC_TIME, sh.EXEC_TIME, CAST(h.TIME_USED / 1000.0 AS DECIMAL(18, 3))) AS EXEC_TIME_MS,
    COALESCE(s.LOGIC_READ_CNT, sh.LOGIC_READ_CNT, h.N_LOGIC_READ) AS LOGICAL_READS,
    COALESCE(s.PHY_READ_CNT, sh.PHY_READ_CNT, h.N_PHY_READ) AS PHYSICAL_READS,
    COALESCE(s.DATA_PAGE_CHANGES_CNT, sh.DATA_PAGE_CHANGES_CNT) AS DATA_PAGES_CHANGED,
    COALESCE(s.UNDO_PAGE_CHANGES_CNT, sh.UNDO_PAGE_CHANGES_CNT) AS UNDO_PAGES_CHANGED,
    COALESCE(s.NET_BYTES_SEND, sh.NET_BYTES_SEND) AS BYTES_SENT_TO_CLIENT,
    COALESCE(s.NET_BYTES_RECV, sh.NET_BYTES_RECV) AS BYTES_RECEIVED_FROM_CLIENT,
    COALESCE(s.NET_ROUNDTRIPS, sh.NET_ROUNDTRIPS) AS ROUNDTRIPS_TO_FROM_CLIENT,
    COALESCE(s.REDO_SIZE, sh.REDO_SIZE) AS REDO_SIZE,
    COALESCE(s.IO_WAIT_TIME, sh.IO_WAIT_TIME) AS IO_WAIT_TIME_MS,
    COALESCE(s.TAB_SCAN_CNT, sh.TAB_SCAN_CNT) AS TAB_SCAN_COUNT,
    COALESCE(s.EXEC_CPU, sh.EXEC_CPU) AS EXEC_CPU_MS,
    h.AFFECTED_ROWS AS ROWS_PROCESSED
FROM (SELECT ? AS EXEC_ID) e
LEFT JOIN V$SQL_STAT s ON s.EXEC_ID = e.EXEC_ID
LEFT JOIN V$SQL_STAT_HISTORY sh ON sh.EXEC_ID = e.EXEC_ID
LEFT JOIN V$SQL_HISTORY h ON h.EXEC_ID = e.EXEC_ID
```

## SQL: Call ET

按 `exec_id` 获取算子级执行跟踪（算子名、耗时、占比、排名、内存、哈希统计等）。ET 无结果时，工具会回退查询 `V$SQL_NODE_HISTORY`。

```sql
CALL ET(?);
```

## SQL: Get Sql Node History By Exec Id

ET 调用失败或无返回行时，按 `exec_id` 查询算子级节点统计（`V$SQL_NODE_HISTORY`）。

```sql
SELECT
    SEQ_NO,
    TYPE$,
    N_ENTER,
    MEM_USED,
    DISK_USED,
    HASH_USED_CELLS,
    HASH_CONFLICT,
    DHASH3_USED_CELLS,
    DHASH3_CONFLICT,
    TIME_USED
FROM V$SQL_NODE_HISTORY
WHERE EXEC_ID = ?
ORDER BY SEQ_NO
```

## SQL: Get Explain Session Parameters

查询与 Hash 聚合、Sort、Hash Join 相关的会话 INI 参数（`V$PARAMETER`）。同一连接池在 5 分钟内复用缓存结果。

```sql
SELECT
    NAME,
    TYPE,
    VALUE,
    SYS_VALUE,
    FILE_VALUE
FROM V$PARAMETER
WHERE NAME LIKE '%HAGR%'
   OR NAME LIKE '%SORT%'
   OR NAME LIKE '%HJ%'
ORDER BY NAME
```

## 输出说明

| 字段 | 说明 |
|------|------|
| exec_id | 本次真实执行的 ID；可在库内 `CALL ET(exec_id)` 复现算子跟踪 |
| origin_plan | 计划树原文（dmPython `Connection.explain`） |
| statistics.statement | 语句级 `logical_reads`、`physical_reads`（含 0） |
| statistics.operators | 以计划行号 `seq` 为键；含 `operator`、`time_us`、`time_percent`、`rank`、`memory_kb`、`disk_kb`、`n_enter`、`hash_used_cells`、`hash_conflict`、`dhash3_used_cells`、`dhash3_conflict`、`hash_same_value`；Sort / Hash Join 等算子可附带 `ini`（参数名 → value） |
