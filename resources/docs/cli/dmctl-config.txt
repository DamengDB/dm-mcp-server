# dmctl 配置（远端 MCP）

可执行文件与 `--help` 的用法见 [使用方式](./usage)。

## 首次配置

当第一个参数是 `setup` 时进入交互向导；该路径不走后续由缓存元数据拼出来的动态子命令，因此不会出现在 `dmctl --help` 列出的子命令里（无缓存时帮助文本为基础占位，有缓存时来自元数据）。

不在帮助里列出 `setup`，是为了减少自动化流程（例如 Agent）误进交互向导、改写连接信息。应在终端由人工执行：

```bash
dmctl setup
```

下面按向导里会出现的项说明用途。

### MCP 服务地址

达梦 MCP 的 HTTP(S) 入口，用于注册会话、列举工具、发起调用。须与实际部署 URL 一致（含协议、主机、端口、路径前缀）。

登录**前端控制台**，打开**首页**，在「CLI 工具服务地址」卡片中可看到当前环境的 MCP 入口；点击**复制**，将剪贴板里的地址粘贴到本向导的「MCP 服务地址」一项即可（下图为该卡片示意，实际 URL 以你环境为准）。

![控制台首页「CLI 工具服务地址」卡片，可复制地址填入 dmctl setup 的 MCP 服务地址](/images/console/cli-mcp-service-address.png)

### 令牌（API Key）

请求里携带的凭证，由服务端校验。申请与轮换按运维或平台文档。不要把密钥写进版本库；向导写入本地缓存后，仍应对该文件限制访问权限。

### 调试模式

是否在标准错误输出更详细的运行日志，用于排查连接、序列化、HTTP 错误。不需要排障时可关闭，以减少日志量与敏感信息外泄风险。

### 截断限制（字符数）

- 对 JSON 里的字符串：长度超过阈值时，标准输出只保留约前 1024 个字符（尽量在换行处截断），并带 `... [完整内容: …]`；完整内容写到「截断文件输出目录」。规则细节见 [输出截断](./output-truncation)。
- 对同时含 `columns` 与 `records` 的对象：按表处理，超过阈值时裁行并写 `_truncated`，整表落盘。见同一文档。

默认一般为 `2048`（以向导提示为准）。设为 `0` 关闭截断，大结果会整段打到标准输出，占用终端与内存，自行权衡。

### 截断文件输出目录

触发截断时完整内容写入的目录，可为相对路径（相对当前工作目录）或绝对路径。默认常见为 `./truncated`（以向导为准）。目录需可写，不存在时程序一般会创建。

向导结束后，在与可执行文件同目录生成 `dmctl-cache.bin`，其中包含连接信息、会话与已下载的命令说明。之后子命令依赖该文件。

- 修改配置：单独改地址、令牌、调试、截断等。
- 更新工具列表：保留连接配置，仅从服务器重新拉元数据与会话。
