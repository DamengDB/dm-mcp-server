CLI 工具使用方式

使用方式

dmctl 与 dmctlx 的调用方式一致：分组、子命令、参数选项，默认在标准输出打印 JSON。首次使用前都要在终端执行各自的 setup，把配置写入与可执行文件同目录的缓存文件（dmctl 为 dmctl-cache.bin，dmctlx 为 dmctlx-cache.bin）。字段说明见 dmctl 配置、dmctlx 配置。

获取可执行文件

从发布页下载对应平台文件，加入 PATH 或使用绝对路径调用。

帮助与子命令

dmctl --version
dmctlx --version
 
dmctl --help
dmctlx --help
 
dmctl <分组> --help
dmctl <分组> <子命令> --help
 
dmctlx <分组> --help
dmctlx <分组> <子命令> --help

dmctl 在生成缓存后，子命令列表来自服务端元数据，可能与文档不同步，以 --help 为准。dmctlx 的子命令随发行版本变化，同样以 --help 为准。

dmctlx 示例

完成 dmctlx 配置 后：

dmctlx query exec_query --sql "SELECT * FROM dual"
dmctlx meta get_db_objects_list --schema SYSDBA --object_type TABLE
dmctlx inspect find_long_active_sessions --threshold_ms 5000

setup 与缓存

两个工具都在首次配置后依赖同目录下的二进制缓存。setup 不在 dmctlx --help 的子命令列表中（实现里为隐藏命令），需要时在终端手动执行。dmctl 侧同理，见 dmctl 配置。
