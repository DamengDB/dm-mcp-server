# dmctlx 配置（本地直连）

可执行文件与 `--help` 的用法见 [使用方式](./usage)。

dmctlx 在本机直连数据库，不经过 MCP Server，也不使用 MCP 侧的 Token。分发形态：Windows 为 `dmctlx.exe`，Linux 为 `dmctlx`（放入 `PATH` 或使用绝对路径）。首次配置在终端执行 `dmctlx setup`，向导把结果写入与可执行文件同目录的 `dmctlx-cache.bin`。

## 首次配置

当第一个参数是 `setup` 时进入交互向导；该路径不走后续由子命令拼出来的动态路由，因此不会出现在 `dmctlx --help` 列出的子命令里。

不在帮助里列出 `setup`，是为了减少自动化流程（例如 Agent）误进交互向导、改写连接信息。应在终端由人工执行：

```bash
dmctlx setup
```

下面按向导里会出现的项说明用途。**调试模式、截断限制、截断文件输出目录** 与 dmctl 向导中的含义、默认行为及安全注意一致，此处不重复展开，请直接阅读 [dmctl 配置](./dmctl-config) 中同名小节（并配合 [输出截断](./output-truncation)）。

### 主机地址

数据库主机名或 IP。

### 端口

数据库监听端口。

### 用户名

登录用户，例如 `SYSDBA`。

### 密码

数据库登录密码；允许空字符串。

### 调试模式

与 [dmctl 配置](./dmctl-config) 中「调试模式」一致。另：环境变量 `DMCTLX_DEBUG` 为 `1`、`true` 或 `yes` 时也会打开调试日志，与缓存里 `debug_mode` 为真时的行为一致（见源码 `main.py`）。

### 截断限制（字符数）

与 [dmctl 配置](./dmctl-config) 中「截断限制（字符数）」及 [输出截断](./output-truncation) 一致。

### 截断文件输出目录

与 [dmctl 配置](./dmctl-config) 中「截断文件输出目录」一致。

向导结束后，在与可执行文件同目录生成 `dmctlx-cache.bin`，其中包含连接信息、调试与截断等配置。之后子命令依赖该文件。

## 已有缓存时的维护菜单

- 修改配置：单独改连接、调试、截断等。
- 重新配置：清空后重新走一遍向导。
- 退出：不保存。
