# 安装与部署

本文说明如何在本地或 Docker 中部署达梦 MCP 服务。

## 环境要求

- **Python** ≥ 3.12
- **[uv](https://github.com/astral-sh/uv)** 包管理器

## 本地安装

### 1. 克隆并准备配置

```bash
git clone <repo-url> dm-mcp-server
cd dm-mcp-server
cp .env.example .env
```

编辑 `.env`，**必须**设置 `APP_SECRET`（长随机字符串）：

```env
APP_SECRET=CHANGE_ME_TO_A_LONG_RANDOM_STRING
```

### 2. 安装依赖

```bash
uv sync
```

### 3. 启动服务

默认 stdio 模式（适合 MCP 客户端以子进程方式拉起）：

```bash
uv run dm-mcp-server
```

HTTP 模式（适合 Web 控制台与远程 AI 客户端）：

```bash
uv run dm-mcp-server --server.transport http --server.host 0.0.0.0
```

等价的入口命令：

```bash
uv run python src/main.py --server.transport http
```

### 4. 验证

```bash
curl -fsS http://localhost:18081/dm-mcp/api/v1/config
```

浏览器访问：

- 控制台：`http://localhost:18081/dm-mcp/`
- 文档：`http://localhost:18081/dm-mcp/docs/`

## Docker 部署

```bash
cp .env.example .env
# 编辑 .env，设置 APP_SECRET

docker compose up --build
```

| 项目 | 说明 |
|------|------|
| 镜像 | `dameng/dm-mcp-server:0.1.0` |
| 入口 | `dm-mcp-server` |
| 端口 | `18081` |
| 配置 | `docker-compose.yml` 通过 `env_file: .env` 加载 |
| 数据卷 | `./data`（SQLite）、`./logs`、`./metrics` |
| 健康检查 | `GET /dm-mcp/api/v1/config` |

### 使用外部达梦数据库

在 `.env` 中修改数据库类型与连接参数：

```env
DATABASE__DB_TYPE=dameng
DATABASE__DAMENG__HOST=192.168.1.100
DATABASE__DAMENG__PORT=5236
DATABASE__DAMENG__USER=SYSDBA
DATABASE__DAMENG__PASSWORD=your-password
```

修改后重新启动容器：

```bash
docker compose up --build
```

## 首次使用

服务启动后，按以下顺序完成初始化：

1. **访问控制台** — 打开 `http://localhost:18081/dm-mcp/`
2. **初始化管理员** — 调用 `POST /dm-mcp/api/v1/auth/admin/init-password` 设置管理员密码
3. **配置数据源** — 在 [数据源管理](./console/datasource) 页面添加达梦数据库连接
4. **创建访问令牌** — 在 [令牌管理](./console/token) 页面创建 MCP Token
5. **配置 AI 客户端** — 参考 [认证与数据源访问](./mcp/auth)

## 传输模式

| 模式 | 配置 | 适用场景 |
|------|------|----------|
| `stdio` | `SERVER__TRANSPORT=stdio`（默认） | Claude Desktop、Cursor 等以子进程方式启动 MCP |
| `http` | `SERVER__TRANSPORT=http` | Web 控制台、远程 HTTP 客户端、Docker 部署 |

HTTP 模式内部使用 MCP Streamable HTTP 协议，端点为 `{base_url}/mcp`（默认 `/dm-mcp/mcp`）。
