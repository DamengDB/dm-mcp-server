令牌管理

访问令牌是 AI 客户端调用达梦 MCP 服务时的身份凭证。本节说明如何创建、配置和管理访问令牌，以及如何通过令牌实现细粒度的数据访问控制。

令牌与 JWT 的区别

在达梦 MCP 中有两种令牌，用途各不相同：

类型用途获取方式
JWT 令牌登录 Web 控制台的身份凭证登录时由 OAuth 或管理员认证颁发
访问令牌AI 客户端调用 MCP 工具的 API 凭证在令牌管理中手动创建

本文档中的”令牌”均指访问令牌。

创建访问令牌

点击列表页上方的”创建令牌”按钮，打开创建对话框。

基础配置

字段说明
令牌名称可读标识，便于区分不同用途的令牌
允许访问的数据源多选，决定该令牌可以操作哪些数据源
默认数据源当 AI 客户端未指定数据源时使用的默认值

安全选项

字段说明
过期时间设置令牌的有效期，到期后自动失效。留空表示长期有效
IP 白名单仅允许列表中的 IP 地址使用该令牌，多个地址用逗号分隔
IP 黑名单禁止列表中的 IP 地址使用该令牌，优先级高于白名单
远程主机可选，绑定该令牌可访问的 SSH 主机

创建完成

点击”创建”后，系统生成令牌，可在界面中点击复制按钮复制到剪贴板中。

令牌列表

已创建的令牌以表格形式展示，每条记录显示名称、关联数据源、过期时间和有效状态。

展开详情

点击表格行可展开详情面板，查看该令牌的完整权限配置：

  • 允许访问的数据源列表
  • 默认数据源
  • 过期时间
  • IP 白名单 / 黑名单
  • 绑定的 SSH 主机

复制令牌

如果需要在其他设备或客户端中使用已有令牌，可以在列表中点击”复制”按钮，将令牌字符串复制到剪贴板。

更新令牌

点击令牌行的”编辑”按钮，可以修改以下配置：

  • 令牌名称
  • 允许访问的数据源范围
  • 默认数据源
  • 过期时间（可延长或提前设为过期）
  • IP 白名单 / 黑名单
  • 绑定的 SSH 主机

编辑保存后立即生效，无需重新创建令牌。

删除令牌

不再使用的令牌应及时删除以降低安全风险。支持单条删除和批量删除。

删除操作不可撤销，删除后使用该令牌的 AI 客户端将无法继续访问 MCP 服务。

安全最佳实践

  1. 按用途分令牌 — 为不同的 AI 客户端或不同的使用场景创建独立的令牌，避免一个令牌被多处共用。
  2. 设置合理的过期时间 — 对于临时使用场景，设置较短的过期时间；对于长期使用的生产令牌，建议设置 3-6 个月的过期时间并定期轮换。
  3. 使用 IP 白名单 — 如果 AI 客户端从固定 IP 或固定网段发起请求，建议配置 IP 白名单进一步缩小攻击面。
  4. 限制数据源范围 — 每个令牌只赋予其必需的最小数据源集合，遵循最小权限原则。
  5. 及时清理失效令牌 — 定期检查令牌列表，删除过期或不再使用的令牌。
