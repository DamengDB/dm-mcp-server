5:Tfb8,# 数据源管理

数据源是达梦 MCP 与数据库建立连接的桥梁。所有 MCP 工具的数据库操作都通过已配置的数据源执行。本节说明如何在控制台中创建、配置和管理数据源。

![数据源管理列表页](/images/console/datasource-list.png)

## 创建数据源

点击列表页上方的"添加数据源"按钮，打开创建对话框。

![添加数据源对话框](/images/console/datasource-create-dialog.png)

### 基础连接信息

| 字段 | 说明 |
|------|------|
| 名称 | 数据源的唯一标识，后续在令牌管理和元数据管理中引用 |
| 部署类型 | **STANDALONE**（单机）或达梦数据库集群类型 |
| 主机 | 数据库服务器地址 |
| 端口 | 数据库监听端口，默认为 5236 |
| 用户名 | 连接数据库使用的数据库用户 |
| 密码 | 对应的数据库用户密码 |

### 高级选项

| 字段 | 说明 |
|------|------|
| DSN | 可选，当使用数据源名连接时填写 |
| 只读模式 | 开启后，该数据源仅允许执行查询类操作，禁止写入 |
| 连接池最小连接数 (minsize) | 连接池保持的最小空闲连接数 |
| 连接池最大连接数 (maxsize) | 连接池允许的最大连接数 |
| 连接超时 (timeout) | 建立连接的最大等待时间（秒） |
| 权重 (weight) | 多数据源负载均衡时的权重分配 |

### 连接测试

填写完成后，建议先点击"测试连接"按钮验证配置是否正确。系统会尝试与目标数据库建立连接并返回测试结果。测试通过后，点击"保存"完成创建。

## 数据源列表

创建后的数据源以表格形式展示，支持以下操作：

### 搜索与筛选

列表顶部提供搜索框，支持按数据源名称、主机地址或用户名进行模糊搜索，方便在数据源较多时快速定位。

### 展开详情

点击表格行可展开详情面板，查看该数据源的实时连接池状态：

- 当前连接数 (size)
- 空闲连接数 (freesize)
- 最小连接数 (minsize)
- 最大连接数 (maxsize)
- 部署类型
- 是否只读
- 负载策略

![数据源详情展开面板](/images/console/datasource-details.png)

## 设置默认数据源

当 AI 客户端调用 MCP 工具但未显式指定数据源时，系统使用默认数据源。在列表中通过下拉选择框可以切换默认数据源。

## 启用与禁用

每个数据源都可以单独启用或禁用：

- **启用** — 数据源可被 MCP 工具正常访问
- **禁用** — 数据源保留配置但不再提供服务，MCP 工具不会使用该数据源建立连接

支持单条操作和批量操作。选中多条记录后，可通过批量操作栏一次性启用或禁用。

![批量操作栏](/images/console/datasource-batch-actions.png)

## 重载数据源

当数据库端发生变更（如用户密码更新、连接参数调整）时，可以通过重载操作使配置生效，无需重启服务。

- **单条重载** — 针对单个数据源刷新连接池
- **全部重载** — 一次性刷新所有数据源的连接池

## 编辑数据源

点击数据源行的"编辑"按钮，打开编辑对话框修改配置信息。编辑后建议重新进行连接测试，确认变更后的配置可以正常连接。

## 删除数据源

不再使用的数据源可以删除。删除操作会同时清理该数据源相关的连接池资源。

支持单条删除和批量删除。删除前请确认：

- 没有访问令牌将该数据源设为默认数据源
- 没有活跃的 MCP 连接正在使用该数据源

## 常用操作建议

1. **生产环境建议使用只读模式** — 除非有明确的写入需求，否则为数据源开启只读模式可以降低误操作风险。
