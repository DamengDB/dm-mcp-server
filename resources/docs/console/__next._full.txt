49:Tfb3,# 功能简介

达梦 MCP 提供基于 Web 的控制台，用于管理数据库连接、访问权限和元数据配置。
登录控制台后，左侧导航栏提供四项核心功能模块。本文档按模块分别说明其用途与操作方式。

![控制台首页全貌，左侧导航栏高亮四项功能](/images/console/console-overview.png)

## 功能模块总览

| 模块 | 用途 | 典型操作 |
|------|------|---------|
| [控制台首页](./dashboard) | 查看系统整体状态，获取服务连接信息 | 查看统计卡片、复制 MCP 服务地址、CLI 工具服务地址和 JWT 令牌 |
| [数据源管理](./datasource) | 配置达梦数据库连接 | 添加、编辑、测试、启用/禁用数据源 |
| [远程主机管理](./ssh-host) | 配置 SSH 远程主机 | 添加、编辑、删除 SSH 主机 |
| [令牌管理](./token) | 管理 AI 客户端访问凭证 | 创建、更新、删除访问令牌，控制数据可见范围 |

## 核心概念

使用控制台之前，建议先理解以下五个核心概念及其相互关系。

### 数据源

**数据源**是指向一个达梦数据库实例的连接配置，包含主机地址、端口、用户名、密码以及连接池参数。

达梦 MCP 的所有数据库操作（查询、巡检、元数据读取等）都通过数据源建立连接后执行。你可以配置多个数据源，分别指向不同的数据库实例或集群。

每个数据源都可以独立启用或禁用。禁用的数据源不会被 MCP 工具使用，但配置信息仍然保留。

### 远程主机（SSH）

**远程主机**是可通过 SSH 协议访问的服务器节点。达梦 MCP 中的部分场景需要直接操作数据库所在主机（例如执行主机级诊断命令），此时需要先配置 SSH 连接信息。

远程主机配置完成后，可以在**令牌管理**中将其与特定访问令牌绑定，从而限制该令牌只能访问指定的主机。

### 访问令牌

**访问令牌**是 AI 客户端（如 Claude Code）调用 MCP 服务时使用的身份凭证。它与控制台登录用的 JWT 令牌不同：

- **JWT 令牌** — 用于登录 Web 控制台，由 OAuth 或管理员认证颁发，具有会话有效期。
- **访问令牌** — 用于 MCP 工具调用，在 AI 客户端配置中使用，可长期有效或设置过期时间。

通过令牌管理，你可以精细控制每个令牌的权限范围：

- 允许访问哪些数据源
- 默认使用哪个数据源
- 允许从哪些 IP 地址发起请求（白名单）
- 禁止从哪些 IP 地址发起请求（黑名单）
- 可以访问哪些 SSH 主机

### 元数据

**元数据**是数据库对象的结构信息，包括 Schema、表、视图、列以及它们的属性。达梦 MCP 的元数据管理不仅提供浏览功能，更重要的是允许你为数据库对象补充**业务注释**。

业务注释直接影响 AI 对数据库的理解。例如，为名为 `T_SALES` 的表添加注释"销售订单主表，包含订单头信息"后，AI 在生成查询时能更准确地选择正确的表和字段。

### 访问策略

**访问策略**决定 AI 是否可以"看到"某个数据库对象。每个对象（Schema、表、视图、列）都可以单独设置策略：

- **允许（allow）** — AI 可以访问该对象，MCP 工具会将其包含在返回的元数据中。
- **拒绝（deny）** — AI 无法访问该对象，MCP 工具会将其从返回结果中排除。
- **继承（inherit）** — 默认状态，继承上级对象的策略。例如，表的策略继承自所在 Schema，列的策略继承自所在表。

通过访问策略，你可以精确控制 AI 可见的数据范围，避免敏感表或列被意外暴露。
