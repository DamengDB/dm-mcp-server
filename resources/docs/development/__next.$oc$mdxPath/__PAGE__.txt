# 扩展开发

本文面向维护者，说明如何扩展达梦 MCP 服务的 Provider、中间件和业务服务。

## 架构分层

```
src/dm_mcp/
├── app/          # MCPServer、GlobalContext、路由组装
├── api/          # HTTP 控制器（REST API）
├── domain/       # 业务服务（domain/*/services/）与 MCP Provider
├── infra/        # 配置、持久化、传输层、HTTP 中间件
└── core/         # MCP 协议抽象、服务注册表、异常体系
```

**Service 层**（`domain/*/services/`）实现业务逻辑与生命周期管理。

**Provider 层**（`domain/mcp/providers/`）将 Service 能力暴露为 MCP Tool / Resource / Prompt，不包含业务逻辑。

**函数式注册**（`server.mcp`）提供类似 FastMCP 的装饰器 API，适合快速原型。

## 扩展 MCP Provider

### 1. 创建 Provider 类

在 `src/dm_mcp/domain/mcp/providers/` 下新建文件，继承 `BaseMCPProvider`：

```python
from dm_mcp.core.mcp import BaseMCPProvider

class MyProvider(BaseMCPProvider):
    def __init__(self, my_service) -> None:
        super().__init__()
        self.my_service = my_service
        self._register_routes()

    def _register_routes(self):
        @self.mcp.tool()
        async def my_tool(param: str):
            """自定义工具"""
            return await self.my_service.do_something(param)
```

需要数据源访问时，继承 `BaseDataSourceMCPProvider`（见 `domain/mcp/providers/base.py`）。

### 2. 注册 Provider

修改 `src/dm_mcp/app/server.py` 的 `_load_mcp_providers()`：

```python
from dm_mcp.domain.mcp.providers.my_provider import MyProvider

def _load_mcp_providers(self) -> None:
    # ... 现有代码 ...
    providers = [
        # ... 现有 Provider ...
        MyProvider(self.context.my_service),
    ]
    self.add_mcp_providers(providers)
```

### MCP Router 装饰器

`MCPRouter` 提供三个装饰器：

| 装饰器 | 用途 | 常用参数 |
|--------|------|----------|
| `@self.mcp.tool()` | 注册工具 | `name`、`description`、`group`、`requires_token_auth` |
| `@self.mcp.resource(uri)` | 注册资源 | `description`、`mime_type` |
| `@self.mcp.prompt()` | 注册提示词 | `name`、`description` |

工具函数参数自动解析为 JSON Schema，返回值序列化为 JSON。

## 扩展 MCP 中间件

中间件采用责任链模式，用于认证、审计、指标等横切关注点。

### 1. 创建中间件

在 `src/dm_mcp/domain/mcp/middleware/` 下创建，继承 `BaseMCPMiddleware`：

```python
from dm_mcp.core.mcp import BaseMCPMiddleware

class MyMiddleware(BaseMCPMiddleware):
    async def on_call_tool(self, call_next, name: str, arguments: dict):
        result = await call_next(name, arguments)
        return result
```

### 2. 注册中间件

修改 `src/dm_mcp/app/server.py` 的 `_load_mcp_middlewares()`：

```python
def _load_mcp_middlewares(self) -> None:
    middlewares = [
        # ... 现有中间件 ...
        MyMiddleware(),
    ]
    self.add_mcp_middlewares(middlewares)
```

HTTP 层中间件（CORS、审计、异常处理等）位于 `src/dm_mcp/infra/middleware/`。

## 扩展业务服务

### 1. 创建服务类

在 `src/dm_mcp/domain/` 对应模块下创建，继承 `BaseService`：

```python
from dm_mcp.core.service.base import BaseService

class MyService(BaseService):
    def __init__(self, settings, dependency_service):
        self.settings = settings
        self.dependency = dependency_service

    async def startup(self):
        pass

    async def shutdown(self):
        pass
```

### 2. 创建服务工厂

```python
from dm_mcp.core.service import ServiceFactory, ServiceMetadata

class MyServiceFactory(ServiceFactory):
    def metadata(self) -> ServiceMetadata:
        return ServiceMetadata(
            name="my_service",
            service_type=MyService,
            dependencies=["dependency_service"],
            priority=50,
        )

    def create(self, settings, **deps) -> MyService:
        return MyService(settings, deps["dependency_service"])
```

### 3. 注册到 GlobalContext

修改 `src/dm_mcp/app/context.py` 的 `_register_builtin_services()`：

```python
def _register_builtin_services(self) -> None:
    builtin_factories = [
        # ... 现有工厂 ...
        MyServiceFactory(),
    ]
    self.registry.register_factories(builtin_factories)
```

可选：在 `GlobalContext` 中添加 `@property` 便捷访问。

## 函数式注册（server.mcp）

适合快速开发和原型，无需创建 Provider 类：

```python
from dm_mcp.app.server import MCPServer

def create_server():
    server = MCPServer()

    @server.mcp.tool()
    async def add(a: int, b: int) -> int:
        """加法运算"""
        return a + b

    return server

if __name__ == "__main__":
    MCPServer.run(create_server)
```

### 常用 API

| API | 说明 |
|-----|------|
| `@server.mcp.tool()` | 注册工具 |
| `@server.mcp.resource(uri)` | 注册资源 |
| `@server.mcp.prompt()` | 注册提示词 |
| `server.mcp.auth` | 认证上下文 |
| `server.mcp.metrics` | 指标上下文 |
| `server.mcp.get_service(name)` | 获取已注册服务 |

函数式注册与 Provider 类注册可同时使用，框架自动合并。

### 模块化组织

大型项目建议按功能域拆分模块，每个模块提供 `register_xxx_module(server)` 函数：

```python
# my_project/modules/calculator.py
def register_calculator_module(server):
    mcp = server.mcp

    @mcp.tool()
    async def add(a: int, b: int) -> int:
        return a + b
```

在 `create_server()` 中调用各模块的注册函数。

## 扩展配置

修改 `src/dm_mcp/infra/config/settings.py` 或在子类中扩展：

```python
from pydantic import Field
from dm_mcp.infra.config.settings import Settings

class MySettings(Settings):
    my_config: str = Field(default="default_value", description="配置说明")
```

创建服务器时传入自定义类：

```python
server = MCPServer(settings_cls=MySettings, context_cls=MyGlobalContext)
```

## 日志与审计

普通日志使用标准库 `logging`，系统自动重定向到 Loguru：

```python
import logging
logger = logging.getLogger(__name__)
logger.info("操作完成")
```

审计日志通过 `AuditMCPMiddleware` 自动记录 MCP 操作。自定义审计可注入 `LoggingService`：

```python
from dm_mcp.domain.system.services.logging import LoggingService

audit_logger = logging_service.get_audit_logger()
audit_logger.info("敏感操作")
```

## 指标

定义指标类并记录：

```python
from dataclasses import dataclass
from dm_mcp.infra.metrics.metrics import metric_field

@dataclass
class MyMetrics:
    call_count: int = metric_field("调用次数", "counter")

metrics = MyMetrics()
metrics.call_count = 1
if self.metrics:
    self.metrics.record(metrics)
```

## 生命周期钩子

```python
server = MCPServer()

@server.on_startup
async def startup_hook():
    logger.info("服务器启动中...")

@server.on_shutdown
async def shutdown_hook():
    logger.info("服务器关闭中...")
```

## 自定义 GlobalContext

```python
from dm_mcp.app.context import GlobalContext
from dm_mcp.infra.config.settings import Settings

class MyGlobalContext(GlobalContext[Settings]):
    def __init__(self, settings: Settings):
        super().__init__(settings)
        self.registry.register_factory(MyServiceFactory())

    @property
    def my_service(self):
        return self.registry.get_service("my_service")
```

## 最佳实践

- **单一职责**：每个 Provider / Service 只负责一个功能域
- **依赖注入**：通过 `GlobalContext` 和 `ServiceRegistry` 管理依赖
- **生命周期**：实现 `startup()` / `shutdown()` 确保资源正确释放
- **配置驱动**：可变参数通过 `Settings` 或 API 管理
- **安全**：涉及数据库和文件操作时进行输入验证与权限检查
